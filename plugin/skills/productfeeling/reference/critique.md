# Critique Flow

Critical review that pushes for **greatness**, not adequacy. Where `audit` checks completeness, `critique` holds the work to a high bar and names the gap between what it is and what it could be. Be honest: "fine" is a finding, not a pass.

## Stance

- **Raise the ceiling, don't just clear the floor.** Passing every checklist item can still be forgettable. Ask: what would make this *memorable*, distinctive, worth talking about?
- **Be specific and opinionated.** Name the single strongest lever and the single weakest link. Vague praise helps no one.
- **Earn the praise you give.** Call out what genuinely works — but only with evidence. Do not soften real problems to be nice.
- **Never confuse polish with feeling.** A beautiful surface with no emotional point of view still fails.

## Rubric (1–5 each)

1. **North-star clarity** — is the intended feeling obvious, singular, and consistent?
2. **Ambition** — is this aiming for great, or settling for acceptable?
3. **Moment design** — are peaks, endings, and recovery designed deliberately (peak–end)?
4. **Copy feel** — does tone match the job and the user's state?
5. **Trust hygiene** — anxiety reduced without false claims?
6. **State completeness** — empty/error/loading/success/recovery covered?
7. **Ethic & agency** — free of coercion and hollow delight; user stays in control?
8. **Handoff readiness** — could Impeccable execute from this alone?

## Steps

1. Read the full target — no critiquing from summary.
2. Score each rubric item with evidence.
3. Name the **one highest-leverage change** that would move this from good to great.
4. Name what already works — with evidence, not flattery.
5. If the intended feeling is muddy, rewrite it in one crisp paragraph.
6. State the **ceiling**: what "great" would look like here, so the bar is explicit.

## Output

Average score, per-item table, honest strengths, fixes ranked by leverage, the "what great looks like" paragraph, and the single change to make first.

Persist to `.productfeeling/reviews/<slug>-critique.md` when available.
