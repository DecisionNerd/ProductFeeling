# Concepts

The ideas and definitions the rest of the handbook builds on. Each concept page gives you a working definition, the evidence behind it, where it shows up in the strategies and Tools, Techniques, and Practices (TTPs), and a **For engineers and agents** section that translates the idea into implementation and review terms—so you can use the vocabulary precisely instead of by vibe. Several pages include Mermaid diagrams for the core process (states, awareness, habit cues, trust movements, agency audit).

Read [Customer Discovery](../discovery/index.md) alongside these pages: concepts name the mechanisms; discovery names *whose* experience you are designing.
