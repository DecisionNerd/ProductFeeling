# Random Flow

Serendipity command. Pick a technique (and, if useful, a target) at random to break tunnel vision and surface feeling work the user wouldn't have thought to ask for.

## When to use

- The user is stuck, exploring, or wants a fresh lens.
- Warm-up or "surprise me" moments.

## Steps

1. **Choose a technique at random** from the catalog ([catalog.md](catalog.md)) — the Technique and Evaluate categories are the best pool (e.g. `peaks-ends`, `friction`, `states`, `persona`, `intensity`, `metaphors`, `trust`).
2. **Choose a target:**
   - If the user named or has an active surface/flow, use it.
   - Otherwise pick a plausible high-value surface from `docs/` context (onboarding, empty state, error, first success, cancellation), or ask for one only if nothing is available.
3. **Announce the roll** — "Rolled: `intensity` on your onboarding empty state" — so the choice is transparent, not arbitrary-feeling.
4. **Run that technique's reference flow** normally.
5. Offer another roll (`random`) or a related deliberate command as follow-up.

## Rules

- Genuinely vary the pick across a session; don't keep landing on the same technique.
- Never randomize into destructive or change-making actions — `random` explores and reviews, it does not mutate the product without the user asking.
