# ProductFeeling Catalog

Techniques available via `/productfeeling <command>`.

| Command | When to use |
|---------|-------------|
| init | No FEELING.md yet; need a durable emotion north star |
| feel | Quick general "how does this feel?" pushback |
| audit | Structured inventory of emotional fit across a surface/flow |
| critique | Scored document or design critique with priority fixes |
| review | Pre-ship go / proceed-with-changes / hold gate |
| map | Journey valence across steps (anxiety ↑/↓, relief, pride) |
| jobs | Emotional jobs before / during / after |
| tone | Microcopy and voice emotional calibration |
| peaks-ends | Identify intentional peaks and the ending residue |
| friction | Catalog friction that drains trust or energy |
| delight | Decide where delight belongs (and must not appear) |
| trust | Trust signals vs anxiety amplifiers |
| states | Empty / loading / error / success feeling design |
| persona | Same surface through different emotional lenses |
| intensity | Too flat / too theatrical calibration |
| anti-patterns | Emotional dark patterns and hollow delight |
| metaphors | Sensory/metaphorical anchors for Impeccable |
| brief | Produce a handoff-ready emotion brief |
| handoff | Route work to Impeccable and/or DocSlime |
| sequence | Playbooks for onboarding, checkout, dashboard, etc. |
| tools | Show this catalog |

## Companion skills

| Skill | Use after |
|-------|-----------|
| Impeccable | `brief`, `handoff` — execute craft |
| DocSlime | `handoff` — persist narrative in docs |
