# Persona Flow

Run the same target through multiple emotional lenses.

## Handbook

Customer Discovery: `handbook/discovery/01-ideal-customer-and-users.md` (ICP vs buyer vs user). Method: `05-interview-method.md`. Graduate via `06-synthesis.md`. Prefer `/productfeeling sequence` **Customer discovery** when profiles are missing or proto-only.

## Steps

1. Select 2–4 personas or segments (include one edge / anxious / power user). Prefer evidence from `docs/strategy/`; label proto-personas explicitly.
2. For each: before/during/after feeling and the most dangerous misread; capture fear / competence / embarrassment.
3. Find collisions — where optimizing for A harms B.
4. Recommend the primary lens and documented exceptions.

## Durable write

ICP and personas belong in **`docs/strategy/`** (DocSlime)—not a second SoT in `.productfeeling/`. Offer `handoff` / `init` to persist. Reviews may go under `.productfeeling/reviews/`.
