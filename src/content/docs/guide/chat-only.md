description: "Useful feeling reviews with zero project setup."

You do not need `docs/`, `.productfeeling/`, or `FEELING.md` for a useful review.

1. Paste a flow, screenshot description, PRD excerpt, or copy deck.
2. Run `/productfeeling feel`, `audit`, `map`, or another technique.
3. The agent treats your chat material as context.
4. When you want lasting narrative, run `/productfeeling init` and prefer project **`docs/`**.

**Never feel blocked:** a missing feeling file must never stop a scoped review.  
**Never feel overwhelmed:** the agent should not dump the full handbook or entire docs tree by default.
