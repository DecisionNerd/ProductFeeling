# Positioning

## Frame

**ProductFeeling** is the agent skill for deciding and reviewing how a product should *feel* — upstream of visual craft, beside durable product docs.

| Companion | Owns |
|---|---|
| **ProductFeeling** | Emotional north star, journey valence, tone/trust/friction/delight policy, emotion briefs |
| **Impeccable** | Visual and interaction craft / execution |
| **DocSlime** | Durable product, strategy, experience, and engineering docs |

## One-liner

Agents that already ship layout need a first-class way to name, test, and hand off *feeling* — without becoming ethics police or replacing craft and docs tools.

## Differentiation (working)

- **Not** a polish skill (Impeccable).
- **Not** a docs framework (DocSlime) — but designed to *work inside* DocSlime trees, especially `docs/strategy/` and `docs/experience/`.
- **Not** clinical psychology or persuasion engineering.
- Feeling context is **insinuated through product docs**, not a conspicuous ProductFeeling-only emotion file.

## Messaging anchors

- Feeling is a product requirement.
- ProductFeeling itself aims for **grounded clarity**, with confidence as the residue (see [`../experience/feeling-north-star.md`](../experience/feeling-north-star.md)).
- One primary feeling; earn delight; buy trust; never manipulate.
- Chat-only and project-anchored both first-class.
- Handbook ships with the skill in chunks; agents load depth selectively.
